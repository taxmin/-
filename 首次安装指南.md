# 首次安装指南

## 📋 安装步骤

### 步骤1：获取项目代码

```bash
git clone <仓库地址>
cd 梦幻手游日常多开
```

### 步骤2：获取模板文件（重要！）

由于 `_templates/` 目录包含核心配置，**不会**上传到 Git 仓库。

**请联系开发者获取 `_templates` 目录**，然后将其放到项目根目录：

```
梦幻手游日常多开/
├── _templates/          ← 从这里获取
│   ├── dx_lib_x64_init.py
│   ├── dx_lib_x86_init.py
│   └── README.md
├── dxGame/
├── app/
├── install_env.bat
└── ...
```

### 步骤3：运行安装脚本

```batch
双击 install_env.bat
```

脚本会自动：
1. ✅ 检测 Python 环境
2. ✅ 安装依赖包
3. ✅ 从 `_templates/` 复制 `__init__.py` 文件
4. ✅ 检查 OCR 环境

### 步骤4：配置本地设置

```batch
复制 config_local.example.py 为 config_local.py
编辑 config_local.py，填写你的 VNC 端口和账号信息
```

### 步骤5：启动程序

```batch
双击 run.bat
```

## ❓ 常见问题

### Q1: 运行 install_env.bat 提示"未找到模板文件"

**原因：** 缺少 `_templates/` 目录

**解决：** 
1. 联系开发者获取 `_templates` 目录
2. 解压到项目根目录
3. 重新运行 `install_env.bat`

### Q2: 我可以自己创建 `__init__.py` 吗？

**可以！** 如果无法获取模板，可以手动创建：

**创建 `dxGame/dx_lib/x64/__init__.py`：**
```python
import sys, os
current_dir = os.path.dirname(__file__)
python_version = f"cp{sys.version_info.major}{sys.version_info.minor}"
pyd_files = [f"dxpyd.{python_version}-win_amd64.pyd", "dxpyd.cp39-win_amd64.pyd", "dxpyd.cp38-win_amd64.pyd"]
dxpyd = None
for pyd_file in pyd_files:
    pyd_path = os.path.join(current_dir, pyd_file)
    if os.path.exists(pyd_path):
        try:
            import importlib.util
            spec = importlib.util.spec_from_file_location("dxpyd", pyd_path)
            dxpyd = importlib.util.module_from_spec(spec)
            spec.loader.exec_module(dxpyd)
            break
        except:
            continue
if dxpyd is None:
    raise ImportError(f"Cannot find dxpyd for Python {python_version}")
```

**创建 `dxGame/dx_lib/x86/__init__.py`：**
```python
import sys, os
current_dir = os.path.dirname(__file__)
```

### Q3: 为什么模板不公开？

模板文件包含项目的核心配置逻辑，由开发者统一管理。这样可以：
- 确保配置的正确性
- 方便开发者更新和维护
- 保护核心实现细节
